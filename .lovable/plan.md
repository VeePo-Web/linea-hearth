

# Fix: Google Sign-In is Broken

## Root Cause

The auth logs show a **400 error: "provider is not enabled"** every time a user clicks "Continue with Google." This is because:

1. The code in `useAuth.tsx` calls `supabase.auth.signInWithOAuth({ provider: 'google' })` directly
2. Lovable Cloud manages Google OAuth through its own module (`lovable.auth.signInWithOAuth`)
3. The required `src/integrations/lovable/` module has never been generated for this project

This means **every Google sign-in attempt fails silently** (or with an error toast).

## Fix Steps

### Step 1: Enable Google OAuth via Lovable Cloud

Use the Configure Social Login tool to generate the `src/integrations/lovable/` module with Google provider support. This creates the managed OAuth integration automatically.

### Step 2: Update `useAuth.tsx`

Replace the `signInWithGoogle` method to use the Lovable Cloud managed module instead of the raw Supabase client:

```typescript
// Before (broken):
const { error } = await supabase.auth.signInWithOAuth({
  provider: 'google',
  options: { redirectTo: `${window.location.origin}/`, ... }
});

// After (working):
import { lovable } from "@/integrations/lovable/index";
const { error } = await lovable.auth.signInWithOAuth("google", {
  redirect_uri: window.location.origin,
});
```

### Step 3: Update `GoogleAuthButton.tsx`

No changes needed -- it already delegates to `useAuth().signInWithGoogle`, so fixing the hook fixes the button everywhere (SignInForm, CreateAccountForm).

## Files Changed

| File | Change |
|------|--------|
| `src/integrations/lovable/` (new) | Auto-generated by Configure Social Login tool |
| `src/hooks/useAuth.tsx` | Update `signInWithGoogle` to use `lovable.auth.signInWithOAuth` |

## Risk: Low
- Only the Google OAuth path changes
- Email/password sign-in is unaffected
- The `GoogleAuthButton` component stays identical visually
